# Security Policy

## Scope

HipAAsynth is a synthetic data generation and fairness testing engine. It does not process, store, or transmit real patient data in any form. All patient records generated by this engine are synthetic and contain no PHI.

Security disclosures relevant to this project include:

- Vulnerabilities in the engine that could cause deterministic outputs to become non-deterministic (undermining reproducibility guarantees)
- Vulnerabilities that could allow arbitrary code execution via crafted input to the engine or polymorphic layer
- Flaws in the fairness metric calculations that could cause systematically incorrect pass/fail determinations
- Dependency vulnerabilities in optional packages (e.g. `fhir.resources`)

Security disclosures that are out of scope for this repository:

- The CAP pipeline (cryptographic certification, Bitcoin anchoring) — maintained separately, not open source
- The `hipaasynth-research` private submodule — report separately to [cody@hipaasynth.com](mailto:cody@hipaasynth.com)

---

## Reporting a vulnerability

Do not open a public GitHub issue for security vulnerabilities.

Report privately to:

**cody@hipaasynth.com**

Include in your report:
- A clear description of the vulnerability
- The component affected (engine, DIF module, polymorphic layer, etc.)
- Steps to reproduce
- Your assessment of impact and severity
- Any suggested remediation if you have one

---

## Response timeline

- Acknowledgment within 48 hours
- Initial assessment within 7 days
- Fix or mitigation timeline communicated within 14 days

For vulnerabilities with direct clinical AI safety implications — findings that could cause a model to appear to pass a fairness audit when it should fail — response will be prioritized and expedited.

---

## Disclosure policy

HipAAsynth follows coordinated disclosure. We ask that you give us reasonable time to address a verified vulnerability before public disclosure. We will credit researchers who report valid findings unless they request anonymity.

---

## A note on methodology vulnerabilities

Because HipAAsynth produces evidentiary artifacts used in clinical AI accountability, methodology vulnerabilities — flaws that cause the audit to produce incorrect results — are treated with the same urgency as code vulnerabilities.

If you identify a flaw in how a population profile is calibrated, how a fairness metric is calculated, or how a polymorphic form represents a patient population, report it here or open a public issue labeled `methodology`. Methodology issues are not security issues in the traditional sense, but they carry real-world consequences for the populations this engine exists to protect.
